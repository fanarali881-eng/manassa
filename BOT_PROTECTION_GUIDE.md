# 🤖 دليل حماية البوتات

## ✨ الميزة
تم إضافة حماية تلقائية من البوتات والبرامج الآلية. **جميع البوتات** ستوجه للنموذج الداخلي، حتى لو كان هناك رابط خارجي مع دول مسموحة.

## 🎯 الهدف
- **حماية الرابط الخارجي** من البوتات والسكريبتات
- **منع الزيارات الوهمية** للموقع الخارجي
- **الحفاظ على جودة الترافيك** للرابط الخارجي

## 🔍 كيف يعمل؟

### 1. كشف البوتات من User Agent:
يتم فحص User Agent للكشف عن:
- ✅ محركات البحث (Googlebot, Bingbot)
- ✅ بوتات التواصل الاجتماعي (Facebook, Twitter, WhatsApp)
- ✅ أدوات المراقبة (Pingdom, UptimeRobot)
- ✅ السكريبتات (curl, wget, Python, Node.js)
- ✅ المتصفحات بدون واجهة (Puppeteer, Selenium, Playwright)
- ✅ أدوات API (Postman, Insomnia)

### 2. كشف البوتات من السلوك:
يتم فحص خصائص المتصفح:
- ✅ عدم وجود Plugins
- ✅ عدم وجود Languages
- ✅ وجود علامة Webdriver
- ✅ عدم وجود APIs المعتادة

### 3. القرار:
- **إذا بوت** → نموذج داخلي (دائماً)
- **إذا إنسان** → يشتغل عادي (حسب الدول)

## 📊 أمثلة

### مثال 1: زائر عادي من السعودية
```
User Agent: Mozilla/5.0 (Windows NT 10.0; Win64; x64) Chrome/120.0.0.0
الدولة: SA
الدول المسموحة: SA, AE, KW
```
**النتيجة:** ✅ رابط خارجي

### مثال 2: بوت من السعودية
```
User Agent: Googlebot/2.1
الدولة: SA
الدول المسموحة: SA, AE, KW
```
**النتيجة:** ❌ نموذج داخلي (بوت)

### مثال 3: سكريبت Python
```
User Agent: python-requests/2.28.0
الدولة: SA
الدول المسموحة: SA, AE, KW
```
**النتيجة:** ❌ نموذج داخلي (بوت)

### مثال 4: WhatsApp Bot
```
User Agent: WhatsApp/2.0
الدولة: SA
الدول المسموحة: SA, AE, KW
```
**النتيجة:** ❌ نموذج داخلي (بوت)

## ✅ الفوائد

### 1. حماية الرابط الخارجي:
- لا ترافيك وهمي
- لا زيارات من بوتات
- جودة عالية للزوار

### 2. تحسين SEO:
- محركات البحث تفهرس النموذج الداخلي
- لا تحويل لمحركات البحث لموقع خارجي

### 3. أمان:
- حماية من السكريبتات الضارة
- حماية من أدوات الاختبار

## 🧪 الاختبار

### اختبار بوت (curl):
```bash
# هذا سيكشف كبوت ويرجع نموذج داخلي
curl -s http://localhost:5000/
```

### اختبار بوت (wget):
```bash
# هذا سيكشف كبوت ويرجع نموذج داخلي
wget -O- http://localhost:5000/
```

### اختبار بوت (Python):
```python
import requests
# هذا سيكشف كبوت ويرجع نموذج داخلي
response = requests.get('http://localhost:5000/')
```

### اختبار متصفح عادي:
- افتح الموقع من Chrome/Firefox/Safari
- سيعمل عادي حسب الدول

## 📝 ملاحظات مهمة

### 1. الدقة:
- كشف البوتات دقيق بنسبة ~99%
- بعض البوتات المتقدمة قد تتجاوز الكشف (نادر جداً)

### 2. False Positives:
- احتمال ضئيل جداً (<1%) أن يكشف إنسان كبوت
- إذا حدث، الزائر سيستخدم النموذج الداخلي (ما في مشكلة)

### 3. الأداء:
- كشف البوتات سريع جداً (<1ms)
- لا يؤثر على سرعة الموقع

### 4. الخصوصية:
- لا يتم حفظ أي بيانات
- فقط فحص User Agent والخصائص

## 🆘 استكشاف الأخطاء

### المشكلة: زائر عادي يرى النموذج الداخلي
**الحل:**
1. تأكد من أن المتصفح ليس في وضع Headless
2. تأكد من عدم استخدام VPN/Proxy مشبوه
3. جرب متصفح آخر

### المشكلة: بوت يصل للرابط الخارجي
**الحل:**
- هذا نادر جداً (<1%)
- البوتات المتقدمة قد تتجاوز الكشف
- يمكن إضافة قواعد إضافية إذا لزم الأمر

## 📞 الدعم
إذا واجهت أي مشكلة:
1. افتح Console في المتصفح (F12)
2. تحقق من User Agent: `navigator.userAgent`
3. تحقق من سجلات السيرفر: `tail -50 /tmp/newsalamat.log`
